# Research: Documentation Standards

**Last Updated:** 2026-01-28  
**Research Scope:** Fundamentals, best practices, highest standards, innovative techniques

## Fundamentals

### Core Concepts

**Documentation Types:**
- **README.md:** Project overview, quick start
- **API Documentation:** Endpoints, parameters, examples
- **Architecture Documentation:** System design, decisions
- **Contributing Guide:** How to contribute
- **Code Comments:** Inline documentation

**Documentation Principles:**
- Write for your audience (developers, users, contributors)
- Keep it up-to-date
- Make it discoverable
- Use examples and code samples

## Best Practices

### README Standards

**Essential Sections:**
- Project description (what it does)
- Installation instructions
- Usage examples
- Project structure
- Contributing guidelines
- License information

**README Quality:**
- Clear, concise writing
- Code examples
- Visual aids (diagrams, screenshots)
- Badges (build status, version, license)

### API Documentation

**OpenAPI/Swagger:**
- Machine-readable API specs
- Interactive API explorer
- Code generation support
- Version management

**Documentation Tools:**
- Swagger UI
- Redoc
- Postman Collections
- API Blueprint

### Code Documentation

**Inline Comments:**
- Explain "why", not "what"
- Document public APIs
- Use JSDoc, docstrings, godoc
- Keep comments up-to-date

## Highest Standards

### Documentation Generation

**Automated Documentation:**
- JSDoc/TypeDoc (JavaScript/TypeScript)
- Sphinx (Python)
- godoc (Go)
- rustdoc (Rust)

**Documentation Sites:**
- GitHub Pages
- Read the Docs
- Vercel/Netlify
- Custom documentation sites

### Architecture Documentation

**Architecture Decision Records (ADRs):**
- Document significant decisions
- Track decision history
- Share context and rationale
- Tools: adr-tools, MADR

**Diagrams:**
- System architecture diagrams
- Sequence diagrams
- Component diagrams
- Tools: Mermaid, PlantUML, draw.io
